Nesta seção, exploraremos os conceitos e a evolução da tecnologia de orientação a objetos, compreendendo sua importância e aplicação no desenvolvimento de software moderno.

## Diferenças entre Paradigmas

Analisaremos as limitações e diferenças entre a programação estruturada e a orientação a objetos, destacando as vantagens e desvantagens de cada paradigma.

## Conceitos Básicos

### Objeto e Classe

- **Objetos:** Instâncias de classes que representam entidades do mundo real.
- **Classes:** Modelos ou "blueprints" para a criação de objetos, definindo seus atributos e comportamentos.

### Métodos e Atributos

- **Métodos:** Funções definidas dentro de uma classe que descrevem os comportamentos dos objetos.
- **Atributos:** Variáveis que armazenam o estado de um objeto.

## Pilares da Orientação a Objetos

### Herança

Permite que uma classe herde propriedades e métodos de outra classe, promovendo a reutilização de código.

### Polimorfismo

Capacidade de objetos de diferentes classes serem tratados como objetos da mesma classe base através de interfaces e métodos sobrescritos.

### Encapsulamento

Esconde os detalhes internos de um objeto e expõe apenas o necessário para sua utilização, promovendo a segurança e a modularidade.

## Relações entre Classes

### Agregação

Relacionamento onde uma classe contém referências a objetos de outras classes, mas sem dependência direta.

### Associação

Relação entre classes onde os objetos interagem entre si, podendo ser unidirecional ou bidirecional.

### Dependência

Indica que uma classe depende de outra para funcionar corretamente, geralmente através de parâmetros de método.
